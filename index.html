AACAT Redesign UX/UI DESIGN PROCESS THE PROBLEM: The clarity of the purpose of the coalition and how it wants to accomplish its goals of fighting human trafficking and helping victims is not clearly found on the home page or the site. We believe people who visit the site may have difficulty finding information easily and quickly. The organization of the information on the website is hard to understand. THE SOLUTION: We plan on helping improve navigation and organization of the site so that things are easier to find. We would like to give clear visuals and concise text of what their mission is. We would like to make information easily accessible to those that are in need. Listed below is the UX/UI Design Process.
EMPATHY MAP: After the evaluation, an empathy map was created to better understand the user needs and goals from the site. The information used to create the empathy map came from potential user interview.
HEURISTIC EVALUATION: First thing that was done was a heuristic evaluation on the original website. This was done to get a better understanding of what elements were good and what elements needed work on. With this evaluation it was determined that the navigation of the site was hard to follow and finding information that a user would seek was not clear.
USER INSIGHT: After the creating an empathy map, we then gained more knowledge of what an actual user might be looking for in the site. This is where the user insight comes in. This helped us have a better understanding of what was needed to make the redesign successful.
PROBLEM STATEMENT: After identifying the user insight, the problem statement was created. After the research from our potential user, the problem was determined about the site and the organization of it.
STORYBOARD: After identifying the problem, a storyboard was created on how a potential user might interatcte with the redesigned site. In this story board, it show the user persona Holly. In this story board, Holly shows just how close to home the need for help in her community sex trafficing is and it shows how she uses her mobile device to access the website for information.
FEATURE PRIORITIZATION MATRIX: After devloping a storyboard, a feature proritization matrix was created to asses the most important features that were needed to be added to the new design.
SITEMAP: A sitemap was created to show how a typical user might interact with the site.
WIREFRAMES: Wireframes were then created for a desktop and a mobile version of the AACAT site.
MID PROTOTYPE: Wireframes were then used to create a mid fidelity desktop and mobile version using Figma.
USER TESTING: The mid fidelity versions were then tested to see what parts of the sites worked well and which parts needed adjusting.
A / B TESTING: A / B testing was also done on the main hero image because the images both were found to both be compelling for the project. With this round of testing it was discovered that both images were impactful and both images should be the hero images. The main site was then created to rotate both images so that they were both able to be used.
HIGH FIDELITY VERSIONS: This concluded the research for the redesign of the ACCAT website and here you can see the home page of both versions.
FINAL THOUGHTS: With all the work that went into the redesign there are still things that wold be some future features that could be added in the future.
